# Безопасность

## Публичный репозиторий

Этот репозиторий содержит публичные материалы NARVE, а не исходный код приложения.

## Сообщение об уязвимости

Не публикуйте сведения об уязвимости, секреты, токены, личные данные или рабочий exploit в обычном Issue.

Предпочтительный порядок:

1. Используйте функцию GitHub «Report a vulnerability», если она доступна в разделе Security.
2. Если приватная отправка недоступна, свяжитесь с владельцем репозитория через профиль GitHub и сообщите только общую тему без технических деталей.
3. Дождитесь приватного канала для полного отчёта.

## Что приложить

- затронутую версию;
- краткое описание влияния;
- условия воспроизведения;
- безопасный proof of concept;
- возможный способ исправления;
- подтверждение, что реальные пользовательские данные не публиковались.

## Данные

Никогда не добавляйте в Issue:

- базы памяти;
- историю диалогов;
- записи голоса;
- скриншоты с приватной информацией;
- ключи;
- токены;
- полные диагностические архивы без очистки.

Публичная политика безопасности будет дополняться по мере приближения первой сборки.
